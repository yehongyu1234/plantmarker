# PlantMarker - 植物位置标注软件

**语言 / Languages:** 中文 · [English](README.en.md)

## 项目概述

PlantMarker 是一款基于 Vue 3 + Electron 的桌面植物位置标注软件，用于标记图片中植物的位置和名称，支持导出标注数据用于机器学习训练。

## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|

| TypeScript | ^5.x | 类型系统 |
| Vite | ^5.x | 构建工具 |
| Pinia | ^2.x | 状态管理 |
| Tailwind CSS | ^3.x | 样式框架 |
| Canvas API | - | 图片标注绘制 |

## 功能需求

### 1. 数据导入

#### 1.1 植物名称导入
- 支持格式：`.txt`（每行一个名称）、`.csv`（第一列）
- 导入后显示为可选择的标签列表
- 支持手动添加/删除/编辑植物名称

#### 1.2 图片文件夹导入
- 选择本地文件夹，自动扫描图片文件
- 支持格式：`.jpg`, `.jpeg`, `.png`, `.bmp`, `.webp`
- 左侧缩略图列表展示所有图片
- 显示标注状态：已标注图片在缩略图左上角显示打勾标识（绿色对勾）
- 标注计数显示在缩略图右下角

### 2. 标注功能

#### 2.1 画布操作
- 图片自适应画布大小
- 支持缩放（滚轮）和平移（空格+拖拽）
- 显示当前缩放比例
- 重置视图按钮

#### 2.2 描边工具
- **自由绘制模式**：鼠标按下拖拽绘制任意曲线
- **自动简化**：释放鼠标后，使用 Ramer-Douglas-Peucker 算法将曲线简化为多边形
- **点模式**：点击添加顶点，双击或闭合完成多边形
- **简化阈值**：可调节（epsilon 参数），控制简化程度

#### 2.3 标注可视化
- **标注区域填充**：使用半透明颜色填充多边形区域（透明度 40%）
- **多边形边框**：使用同色系实线描边（线宽 2px）
- **颜色分配**：根据植物名称自动分配固定颜色（哈希取色），确保同一植物名称在所有图片中颜色一致
- **选中状态**：选中标注时边框加粗、填充透明度增加
- **标注标签**：多边形左上角显示植物名称小标签

#### 2.4 标注管理
- 选择植物名称后进行描边
- 每个标注包含：
  - 植物名称（从导入列表选择）
  - 多边形坐标点集合
  - 标注颜色（按植物名称自动分配）
- 支持选中、移动、删除标注
- 支持编辑标注的植物名称
- 标注列表面板，显示当前图片所有标注

#### 2.5 快捷键
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Z` | 撤销 |
| `Ctrl+S` | 保存当前图片标注 |
| `Delete` | 删除选中标注 |
| `Space + 拖拽` | 平移画布 |
| `滚轮` | 缩放画布 |
| `1-9` | 快速选择植物名称 |

### 3. 导出功能

#### 3.1 单张导出 JSON

```json
{
  "image": {
    "filename": "plant_001.jpg",
    "width": 1920,
    "height": 1080
  },
  "annotations": [
    {
      "id": "uuid-xxx",
      "plant_name": "向日葵",
      "polygon": {
        "normalized": [
          [0.123, 0.456],
          [0.234, 0.567],
          [0.345, 0.678]
        ],
        "pixel": [
          [236, 492],
          [449, 612],
          [662, 732]
        ]
      },
      "area_normalized": 0.056,
      "bbox_normalized": [0.123, 0.456, 0.222, 0.222]
    }
  ],
  "metadata": {
    "created_at": "2024-01-01T12:00:00Z",
    "tool_version": "1.0.0"
  }
}
```

#### 3.2 批量导出 JSONL

每行一个 JSON 对象，用于训练数据集：

```jsonl
{"image":"plant_001.jpg","width":1920,"height":1080,"annotations":[{"plant_name":"向日葵","polygon_normalized":[[0.123,0.456],[0.234,0.567],[0.345,0.678]],"bbox_normalized":[0.123,0.456,0.222,0.222]}]}
{"image":"plant_002.jpg","width":3000,"height":2000,"annotations":[{"plant_name":"玫瑰","polygon_normalized":[[0.1,0.2],[0.2,0.3],[0.3,0.4]],"bbox_normalized":[0.1,0.2,0.2,0.2]}]}
```

#### 3.3 归一化说明

**所有坐标均采用归一化处理**（0-1 范围），原因：
1. 不同图片尺寸不同，归一化后统一标准
2. 训练时模型输入通常需要归一化坐标
3. 便于数据增强和迁移学习

归一化公式：
```
x_normalized = x_pixel / image_width
y_normalized = y_pixel / image_height
```

导出同时保留像素坐标，方便人工检查。

## 核心算法

### Ramer-Douglas-Peucker 线条简化

```
输入: 点集 points[], 阈值 epsilon
输出: 简化后的点集

1. 找到距首尾连线最远的点 dmax
2. 如果 dmax > epsilon:
   - 递归简化前半部分
   - 递归简化后半部分
   - 合并结果
3. 否则:
   - 只保留首尾两点
```

### 植物名称颜色分配

```typescript
// 根据植物名称生成固定颜色（HSL）
function getPlantColor(name: string): string {
  let hash = 0
  for (let i = 0; i < name.length; i++) {
    hash = name.charCodeAt(i) + ((hash << 5) - hash)
  }
  const hue = Math.abs(hash) % 360
  return `hsla(${hue}, 70%, 50%, 0.4)`  // 40% 透明度填充
}
```
